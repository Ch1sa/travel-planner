# travel-planner

AIと一緒に旅行計画を作って、Markdownで管理するためのリポジトリ。

旅行1件 = ファイル1個（`trips/YYYY-{slug}.md`）。テンプレートに沿った構造化されたMarkdownを、Claude Codeに埋めてもらう運用です。

## 構成

````
travel-planner/
├── CLAUDE.md                  # Claude Code向け運用ルール
├── templates/
│   └── trip-template.md       # 旅行計画の雛形
└── trips/
    └── YYYY-{slug}.md         # 旅行1件
````

## 使い方

### 1. 新規旅行を作成

````bash
cp templates/trip-template.md trips/2026-taiwan.md
````

ファイル名は `YYYY-{slug}.md` 形式（例: `2026-taiwan.md`, `2026-okinawa.md`）。

### 2. Claude Codeに条件を伝えて埋めてもらう

````bash
claude
````

以下のテンプレを使って条件を投げる。`{}` の部分を書き換え、不要な行は削除。

````
trips/{YYYY-slug}.md を埋めてください。

## 基本条件
- 出発地: {例: 高松空港 / 東京駅}
- 期間: {例: 2泊3日 / 金〜日}
- 目的地: {例: 台南メイン + 高雄サブ}
- 人数: {例: 1人 / 2人}
- 予算目安: {例: 1人15万円程度}

## 旅行スタイル
- {例: 街歩き、グルメ、夜市、写真映え}
- {例: のんびり温泉、自然、絶景}

## 興味・希望
- {例: 歴史的な街並みが好き}
- {例: ローカルな食堂で食べたい}
- {例: 朝が苦手なので午前は遅め}

## 制約・避けたいこと
- {例: 子連れNG / 早朝起き不可 / 雨天時の代替案も}
- {例: 海産物アレルギーあり}

## 3パターンの方向性
- パターンA: バランス重視（推奨案）
- パターンB: {例: メイン都市に連泊、サブ都市は日帰り}
- パターンC: {例: 拠点を入れ替え、サブ都市メインで}
````

### 3. 部分的に編集する

セクション単位で追加・修正を依頼できます。以下はよく使う依頼テンプレ。

**追記する**

````
trips/{YYYY-slug}.md の「{章番号}. {章タイトル}」に {追加内容} を追加してください。
他のセクションは変更しないこと。
````

**書き換える**

````
trips/{YYYY-slug}.md のパターン{A/B/C} を「{現在の内容}」から「{新しい内容}」に変更してください。
旅程表・おすすめポイント・注意点もそれに合わせて書き直すこと。
````

**1パターンだけ詰め直す**

````
trips/{YYYY-slug}.md のパターン{A/B/C} の旅程表を、以下の条件で詰め直してください。
- {条件1: 例「初日は午後着なので午前の予定は不要」}
- {条件2: 例「2日目に〇〇を必ず入れる」}
他のパターンは変更しないこと。
````

**グルメ・豆知識だけ強化**

````
trips/{YYYY-slug}.md の「7. おすすめグルメ」と「8. 豆知識・トリビア」を、
{エリア名} について追加で {N} 件ずつ充実させてください。
旅程表・予算欄は変更しないこと。
````

### 4. 確認する

- **ローカル**: VS Code/Cursorのプレビュー（`Ctrl+Shift+V`）
- **GitHub**: pushして該当ファイルを開く。表・画像が綺麗にレンダリングされる
- **スマホ**: GitHubアプリ or ブラウザでリポジトリにアクセス

### 5. PDFにする（同行者用パンフレット）

GitHubで該当ファイルを開いて、ブラウザの印刷 → PDFに保存。

## テンプレートの構成

`templates/trip-template.md` は以下の10章構成。

1. 提案の結論（推奨パターンの早見表）
2. 旅行全体の概要
3. フライト・移動の目安
4. **旅程表（3パターン必須：A推奨 / B / C）**
   - 各パターンに時刻入り旅程・おすすめポイント・注意点
5. 予算目安
6. おすすめ観光場所まとめ
7. おすすめグルメ
8. 豆知識・トリビア
9. このプランのメリット・注意点
10. 参考情報・確認先

詳細は `CLAUDE.md` を参照。

## 運用ルール

- 旅行1件 = ファイル1個
- ファイル名: `YYYY-{slug}.md`
- 編集はセクション単位で（Claude Codeに `str_replace` で依頼）
- 章番号と見出しレベルは崩さない
- 旅程表は必ず3パターン作成
- 予約前には公式サイトで最新情報を確認
